# freeBoxとは

freeBox は、hsBox を拡張するための仕組みとして検討している構想です。  
完成した製品や確定した機能ではなく、「どのように拡張できるか」「どのように関われるか」を考えるための枠組みとして位置づけています。

現在は構想・検討段階であり、内容は今後の議論や検証によって変更される可能性があります。

---

## freeBoxの位置づけ

hsBox は、日本国内向けに提供する安定した製品・仕組みを前提としています。  
一方で、利用者の環境や関心、技術レベルはさまざまであり、すべての機能を一つの完成形にまとめることには限界があります。

freeBox は、hsBox 本体とは切り離された形で、

- 機能を追加・拡張する
- 試行的な仕組みを検討する
- 利用者や開発者が関われる余地を残す

といった目的のための「拡張のための箱」として考えています。

---

## なぜ freeBox を考えているのか

hsBox を使う人は、

- 家庭ユーザー
- スマートホーム初心者
- 技術に詳しい人
- 特定の用途だけを求めている人

など、非常に幅広い層に分かれます。

すべての人に向けて「最初から全部入り」の構成を用意するよりも、  
**必要な機能を、必要な人が選んで追加できる構造**のほうが現実的ではないかと考えました。

freeBox は、そのための考え方や仕組みを整理するための取り組みです。

---

## ミニモジュールという考え方

freeBox では、比較的小さな単位の機能を「ミニモジュール」として扱うことを想定しています。

- ひとつの役割に絞った機能
- 組み合わせて使うことを前提とした設計
- GitHub 上で管理・共有できる形

現時点では、名称や具体的な実装方法も含めて検討中であり、  
「このような粒度で考えたい」という方向性を示すものです。

---

## 現在のステータス

freeBox は、現在次のような段階にあります。

- 構想・設計検討段階
- 仕様や名称は未確定
- 実装方法も含めて検討中

そのため、このページに記載されている内容は、  
今後の議論や試作によって変更・更新されることがあります。

---

## 今後について

今後は、以下のような点について順次整理・検討を進めていく予定です。

- GitHub を前提とした公開・管理方法

これらの情報は、GitHub 上で順次公開・更新していきます。
