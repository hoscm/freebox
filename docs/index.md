# freeBoxとは

freeBox は、hsBox を拡張するための仕組みとして検討している構想です。  
完成形を最初に決め切るのではなく、「どのように拡張できるか」「どのように関われるか」を考えるための枠組みとして位置づけています。

その最初の実装として **freeBox Loader v1.0.0** をリリースしました。  
構想全体は引き続き発展中であり、内容は今後の議論や検証によって追加・変更される可能性があります。

---

## freeBoxの位置づけ

hsBox は、日本国内向けに提供する安定した製品・仕組みを前提としています。  
一方で、利用者の環境や関心、技術レベルはさまざまであり、すべての機能を一つの完成形にまとめることには限界があります。

freeBox は、hsBox 本体とは切り離された形で、

- 機能を追加・拡張する
- 試行的な仕組みを検討する
- 利用者や開発者が関われる余地を残す

といった目的のための「拡張のための箱」として考えています。

---

## なぜ freeBox を考えているのか

hsBox を使う人は、

- 家庭ユーザー
- スマートホーム初心者
- 技術に詳しい人
- 特定の用途だけを求めている人

など、非常に幅広い層に分かれます。

すべての人に向けて「最初から全部入り」の構成を用意するよりも、  
**必要な機能を、必要な人が選んで追加できる構造**のほうが現実的ではないかと考えました。

freeBox は、そのための考え方や仕組みを整理するための取り組みです。

---

## Module という考え方

freeBox では、比較的小さな単位の機能を **Module** として扱うことを想定しています。

- ひとつの役割に絞った機能
- 組み合わせて使うことを前提とした設計
- GitHub 上で管理・共有できる形

v1.0.0 では Module を Manager UI から選択・導入・削除する仕組みを **freeBox Loader** として実装しました。  
Module の粒度や運用ルールについては、実装と利用を通じて引き続き整理していく方針です。

---

## 現状の進捗と今後について

freeBox は、現在次のような段階にあります。

- **freeBox Loader v1.0.0 リリース**（hsBox 上で Module を管理する基盤の最初の実装）
- 構想全体としての枠組み整理は継続中
- 利用と検証を通じて仕様・名称を調整していく前提

そのため、このページに記載されている内容は、  
今後の議論や試作によって変更・更新されることがあります。

今後は、以下のような点について順次整理・検討を進めていく予定です。

- freeBox および関連要素のネーミング（v1.0.0 では `freeBox` / `freeBox Loader` / `Module` / `Plugin` を採用済み）
- Module の構成や考え方（v1.0.0 で基本構造は確立。粒度・運用は引き続き検討）
- GitHub を前提とした公開・管理方法（v1.0.0 で公開・配信ルートを整備済み）
- 関心のある方が関われる形の検討

これらの情報は、GitHub 上で順次公開・更新していきます。
