## はじめに

本書は、hoscm が提供する製品およびサービスとサードパーティとのコラボレーションについて、その形態・条件・運用ルールを定めるものです。本プログラムは、初版（Rev.1.0）において freeBox を対象とし、将来的には hoscm が扱う他の製品およびサービスへも順次拡張する予定です。

本プログラムは、参加者の予見可能性を高め、健全で持続的なコラボレーション環境を整備することを目的とします。記載内容は将来にわたり改訂される可能性があり、改訂内容は本書の改訂履歴に記録します。

## 第1章　総則

### 1.1　本プログラムの目的

本プログラムは、hoscm の製品およびサービスとサードパーティとの協業を、形態に応じて整理し、参加者にとって透明かつ予見可能な参加条件を提供することを目的とします。具体的には、コラボレーションの分類、申請・審査の流れ、知的財産・ブランドの取扱い、有償支援の範囲などを定めます。

### 1.2　適用範囲

本プログラムの適用対象は次の通りです。

- 初版（Rev.1.0）の対象：freeBox
- 将来の対象：hoscm が扱う製品およびサービス（拡張は要望および市場動向により順次実施）

hsBox など freeBox 以外を対象とするコラボレーションについては、別途の取扱いを定めるか、本プログラムを拡張する形で対応します。

### 1.3　用語の定義

本書において、以下の用語は次の意味で使用します。

| 用語 | 定義 |
|------|------|
| 参加者 | 本プログラムに基づき hoscm とコラボレーションを行う、または行おうとする者。法人・個人を問わない。 |
| プラグイン | freeBox Loader 上で動作する拡張モジュール。 |
| 公開ステータス | プラグインの公開状態を示す値。public／restricted／private を含む。 |
| 公開仕様 | hoscm が GitHub および公式 Web サイト等で公開しているドキュメントおよび仕様。 |
| サンドボックス環境 | hoscm が管理する、プラグイン検証のための隔離環境。 |

### 1.4　言語と免責

本書の正式な言語は日本語とします。英語表記は参考訳として用意される場合がありますが、解釈に相違が生じた場合は日本語版を優先します。翻訳の不正確さに起因するトラブルについて、hoscm は責任を負いません。

### 1.5　ライセンス

freeBox プロジェクトで提供されるコードは、原則として GNU General Public License v3.0（GPLv3）の下で提供されます。GitHub リポジトリにライセンス全文が掲載されています。

### 1.6　ロードマップに関する注記

freeBox に関するロードマップは参考情報であり、将来の実装を保証するものではありません。事前の告知なく変更または中止される場合があります。一方、hsBox に関する未公開仕様およびロードマップは、原則として NDA の対象として取り扱います（第5章を参照）。

## 第2章　コラボレーションパターン

### 2.1　パターン一覧

freeBox に関するコラボレーションは、hoscm の関与度に応じて次の5つのパターンに分類されます。実装結果として公開されるプラグインの公開ステータスは、いずれのパターンであっても適用される独立した軸として扱います（第4章を参照）。

| 区分（名称） | 内容 | hoscm の関与 | 費用 |
|---|---|---|---|
| 提供元単独実装 | 公開仕様に基づき、参加者が単独で実装する | なし（公開ドキュメントのみ） | 無料 |
| 技術情報提供 | 個別の技術情報を提供し、参加者が実装する | 情報提供 | 有償サポート |
| 技術支援 | 設計・実装支援を提供し、参加者が実装する | 設計・実装支援 | 有償技術支援 |
| 共同開発 | freeBox／hsBox 側の機能強化を含む双方向の開発を行う | 双方向 | 個別見積 |
| 受託開発 | 特定参加者向けの個別仕様を hoscm が実装する | hoscm が実装 | 業務委託（個別見積） |

「受託開発」は厳密にはコラボレーションではなく業務委託に該当しますが、参加者からの問い合わせ範囲の明確化を目的として本表に含めます。

### 2.2　提供元単独実装

参加者が、GitHub および公式 Web サイト等で公開されている仕様・ドキュメントのみを参照して、単独でプラグインまたは連携機能を実装する形態です。

- hoscm からの個別支援は提供されません。
- 参加費は無料です。
- 実装結果を public または restricted で公開する場合は、第3章および第4章の規定に従い登録および審査を経る必要があります。
- private（ローカル利用）の場合は登録不要です。

### 2.3　技術情報提供

参加者の実装に対し、hoscm が個別の技術情報（仕様の補足説明、実装ヒント、検証パターン等）を提供する形態です。実装作業は参加者が行います。

- 提供内容は問い合わせベースとなります。
- 有償サポートとして対応します（料金は個別見積）。
- 提供した技術情報の取扱いについて、必要に応じて NDA を締結します。

### 2.4　技術支援

参加者の実装に対し、hoscm が設計レビュー・実装支援・検証支援などの実作業を伴う支援を提供する形態です。

- 有償技術支援として対応します（料金は個別見積）。
- 実作業の範囲・期間・成果物は個別契約により定めます。

### 2.5　共同開発

参加者と hoscm が双方の機能強化を伴う共同開発を行う形態です。freeBox／hsBox 側の機能拡張も対象になり得ます。

- 共同開発の対象範囲・知的財産の取扱い・成果物の公開条件等は、個別契約により定めます。
- hsBox 側の機能や非公開仕様が含まれる場合は、原則として NDA を締結します。

### 2.6　受託開発

参加者が定める個別仕様に基づき、hoscm がプラグインまたは連携機能を実装する形態です。

- 業務委託契約として対応します。
- 当該パターンは厳密にはコラボレーションには含まれませんが、本書では参加者の問い合わせ範囲の明確化のために掲載しています。

## 第3章　申請と審査

### 3.1　申請窓口

本プログラムへの参加申請は、以下の Web フォームより受け付けます。

- 申請フォーム：https://hoscm.com/hsbox/hoscm-collabo/

申請内容には、コラボレーションパターン、対象プラグインまたは連携対象、想定する公開ステータス、概要などを記載します。

### 3.2　参加費

本プログラムへの参加自体は無料です。ただし、コラボレーションの内容によっては有償の支援が発生します。

無料の範囲：

- Web フォームによる参加申請
- 公開仕様の利用
- GitHub リポジトリの参照
- 公式ドキュメントの利用
- プラグインの公開申請

有償の範囲：

- 個別の技術相談（技術情報提供）
- 技術支援・実装支援
- 検証および認証作業
- 共同開発
- 受託開発

### 3.3　審査プロセス

本プログラムは「利用者ファースト」と「運用コストの最小化」を両立する方針を採ります。プラグインの公開ステータスに応じて、必要な審査の範囲を変えます。

審査の段階は次の通りです。

1. 申請受付（Web フォーム）
2. 基本審査（適合性確認）
3. 技術審査（公開ステータスに応じて実施）
4. 公開・登録（GitHub への掲載）

審査基準は次の観点を含みます。

- 安全性
- 互換性
- 透明性
- 利用者にとっての価値

### 3.4　プラグイン公開時の審査チェックリスト

プラグインを public または restricted で公開する際の審査チェックリストは次の通りです。詳細な運用基準は別途整備します。

**基本情報**

- プラグイン名称が明確である
- 開発者情報（参加者）が記載されている
- 利用目的が明確である

**技術要件**

- freeBox 仕様に準拠している
- エラー処理が適切に行われている
- 不正アクセスを誘発しない設計である
- 不要な外部通信を行わない

**セキュリティ**

- 悪意のあるコードを含まない
- 機密情報を不当に送信しない
- 既知の脆弱性が確認されていない

**品質**

- 動作検証ログが提出されている（restricted 申請時）
- ドキュメントが整備されている
- インストール手順が明確である

**法的要件**

- ライセンスが明記されている（原則 GPLv3）
- 第三者の権利を侵害していない

## 第4章　プラグイン公開ステータス

### 4.1　公開ステータスの定義

freeBox Loader が認識するプラグインの公開ステータスは次の通りです。

| ステータス | 表示ラベル | 表示色 | deploy ボタン挙動 | 確認ダイアログ |
|---|---|---|---|---|
| public | 公開済み | 緑 | 通常ボタン | なし |
| restricted | 限定公開 | 黄 | 通常ボタン | 1段階確認 |
| private | 非公開・ローカル | 赤 | 警告ボタン | 2段階確認（同一モーダル内） |

### 4.2　将来追加予定のステータス

将来的に、以下のステータスを追加する方針です。追加時期および表記は別途定めます。

- official：hoscm 公式提供のプラグイン
- archived：提供を終了したプラグイン

### 4.3　ステータス別の審査基準

公開ステータスごとの審査基準は次の通りです。

| ステータス | 審査内容 |
|---|---|
| public | サンドボックス評価＋実機検証＋複数ユーザーによる評価結果に基づく総合判断 |
| restricted | 書類審査＋セルフテストログの提出 |
| private | 審査不要（ローカル利用のみ） |

### 4.4　検証方針

プラグインの検証は、hoscm が管理するサンドボックス環境および将来構想として整備する分散検証の枠組みにより実施します。

- 内部検証：hoscm 管理のサンドボックス環境で評価
- 実運用検証：信頼できる hsBox ユーザー環境（複数）から取得した検証ログを用いた評価
- 標準化：検証パターンを標準として提供する方針

サンドボックス環境は hoscm が管理する内部評価用であり、外部への提供は行いません。外部から評価依頼があった場合は、有償の支援として対応します。

## 第5章　知的財産・NDA

### 5.1　ライセンスの取扱い

freeBox に関するコードは、原則として GPLv3 の下で提供および授受されます。GitHub リポジトリのライセンス全文が正式な条件です。

参加者が公開するプラグインについても、原則として GPLv3 を採用することを推奨します。READMEファイルおよび LICENSE ファイルの記載例は、リポジトリのテンプレートとして提供します。

### 5.2　NDA 適用方針

freeBox はオープン仕様を基本としており、原則として NDA は不要です。一方、hsBox の内部情報や非公開仕様が含まれる場合は、原則として NDA を締結します。

| ケース | freeBox | hsBox |
|---|---|---|
| 未公開仕様・ロードマップ | 原則不要（参考情報として提供） | 必須 |
| hsBox との連携検討 | 対象外 | 必須 |
| セキュリティ設計の共有 | 必要に応じて締結 | 必須 |
| OEM 交渉 | 条件により締結 | 必須 |
| 共同開発契約 | 原則不要（GPLv3） | 必須 |
| 脆弱性情報の共有 | 非公開情報のみ締結 | 必須 |

### 5.3　脆弱性情報の取扱い

公開済みの OSS および公開済みの脆弱性情報については、NDA は不要です。未公開の脆弱性情報を共有する場合に限り、必要に応じて NDA を締結します。

## 第6章　ブランド・ロゴ利用規定

### 6.1　対象

本規定の対象は次の通りです。

- freeBox
- hsBox
- hoscm

### 6.2　表記ルール

正式表記は次の通りです。大文字・小文字を含めて厳格に守ってください。

| 名称 | 正式表記 | 注意点 |
|---|---|---|
| freeBox | freeBox | 「f」は小文字、「B」は大文字 |
| hsBox | hsBox | 「h」「s」は小文字、「B」は大文字 |
| hoscm | hoscm | すべて小文字 |

URL など技術的事情で大文字小文字の指定に従えない場合のみ、例外を認めます。

公式ロゴ画像を使用する場合は、縦横比および色を変更しないでください。HEX 値による色定義は本書では行わず、公式ロゴ画像の使用を原則とします。

### 6.3　許可される表示例

参加者は、自社の製品・サービスの説明等に次のような表示を使用できます。

- 「freeBox 対応」
- 「freeBox 連携」
- 「freeBox プラグイン」
- 「Powered by freeBox」

### 6.4　禁止事項

次の行為は禁止します。

- 公式ロゴの改変
- 「公式」「認定」など、hoscm の認定を受けたかのような無断表記
- hoscm が保証していると誤認させる表現
- 違法または公序良俗に反する用途
- 第三者の権利を侵害する用途

### 6.5　利用フロー

ブランドおよびロゴの利用は、本規定に従う場合に限り、事前申請なく使用できます。違反が認められた場合、hoscm は修正または使用停止を求めることがあります。事後是正型の運用とすることで、参加者および hoscm 双方の運用負荷を軽減します。

## 第7章　有償技術支援

### 7.1　支援メニュー

有償技術支援は、内容に応じて次のメニューに分類されます。各メニューの料金は、内容に応じて個別見積となります。

| 区分 | 内容 |
|---|---|
| 技術相談 | スポットでの技術質問への対応 |
| 設計レビュー | 仕様および設計の確認・助言 |
| 実装支援 | 実装作業に対する技術サポート |
| 検証支援 | 動作検証およびログ評価 |
| 受託開発 | 個別仕様の開発 |

### 7.2　価格設定方針

支援内容は案件ごとに大きく異なるため、標準単価は設定しません。すべて個別見積として、内容・期間・成果物の範囲に応じて価格を提示します。

## 第8章　セキュリティと脆弱性管理

### 8.1　基本方針

hoscm は、freeBox および周辺プラグインの安全性確保を最優先とします。脆弱性に関する情報は、hoscm および参加者の双方で適切に管理することを基本方針とします。

### 8.2　報告窓口

脆弱性情報の報告窓口は、当面の間、既存のお問い合わせ窓口を兼用します。専用窓口の設置は、必要性が確認された段階で検討します。

将来にわたって専用窓口の設置が必要となるような事態を生じさせないよう、高品質な製品づくりに注力します。

### 8.3　CVE 採番に関する方針

当面、独自の CVE 採番は行いません。重大な脆弱性が確認された場合は、必要に応じて JPCERT/CC、IPA など、公的な機関と連携します。

OSS 依存ライブラリに関する脆弱性は、各 OSS プロジェクトが管理する CVE 情報を参照します。

## 第9章　拡張・将来計画

### 9.1　拡張方針

本プログラムは、hoscm の製品およびサービスの拡張に応じて、対象範囲を順次拡大します。拡張対象として現在検討されている項目には、hsBox を対象としたコラボレーション、情報発信領域での連携プログラムの整備などが含まれます。

### 9.2　優先順位の見直し

拡張対象および優先順位は、市場動向および参加者からの要望に基づき、適宜見直します。最新の予定は本書の改訂または公式 Web サイトでの告知により周知します。
